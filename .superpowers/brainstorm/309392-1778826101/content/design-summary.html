Schema changes

New table run_logs — one row per log line emitted during a run:

CREATE TABLE run_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  run_id INTEGER NOT NULL REFERENCES runs(id),
  level TEXT NOT NULL, -- INFO / WARNING / ERROR
  message TEXT NOT NULL,
  logged_at TEXT NOT NULL DEFAULT (datetime('now'))
);

Log capture in Runner

Before calling monitor.fn(page, ctx), attach a custom logging.Handler to the monitor's logger that buffers records. After the run completes (ok or error), flush buffered lines to run_logs using the newly inserted run_id.

New routes

GET /monitors/{name} — detail page: last 50 runs + expandable logs per run
GET /activity — global feed: last 50 runs across all monitors, filterable
GET /api/monitors/{name}/runs — JSON: runs + logs, used by poll-after-trigger

Dashboard change

Monitor names become links: <a href="/monitors/{{ m.monitor_name }}">. Add an "Activity" link in the header.

Run now → poll flow

Detail page: click "Run now" → POST /monitors/{name}/run → JS polls GET /api/monitors/{name}/runs every second until a new run_id appears → re-renders the top row with status + logs.

New templates

  • templates/monitor_detail.html — detail page
  • templates/activity.html — global feed
